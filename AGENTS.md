# Agent Guidelines for terraform-provider-airbyte

This file contains guidance for AI coding agents working on this repository. For the full contributor guide (including code generation lineage, release process, and documentation), see [CONTRIBUTING.md](CONTRIBUTING.md).

## Key Principle: This Is a Generated Codebase

Most code under `internal/provider/` and `internal/sdk/` is generated by Speakeasy from an OpenAPI spec. **Do not hand-edit generated files** — your changes will be overwritten on the next regeneration.

To make durable changes, work at the correct layer:

| Change type | Where to make it |
|---|---|
| API behavior | Upstream [OpenAPI spec](https://github.com/airbytehq/airbyte-platform/blob/main/airbyte-api/server-api/src/main/openapi/api.yaml) |
| Terraform schema tweaks | [Overlay](overlays/terraform_speakeasy.yaml) (`x-speakeasy-plan-modifiers`, nullable fields, etc.) |
| Spec transformation | [Generation script](scripts/generate_terraform_spec.py) |
| Post-generation patches | Scripts in `scripts/` wired through `poe_tasks.toml` `post-generate` task |
| Custom plan modifiers | `internal/planmodifiers/` (not generated — safe to edit) |

## Debugging Zero-Diff CI Failures

When the **Zero-Diff Check (Generated Code)** CI job fails, it means your committed code doesn't match what Speakeasy generates. **Do not guess or iterate blindly.** Download the generated artifacts from CI and diff them.

```bash
# 1. Find the CI run ID for your PR
gh run list --workflow="test-full.yml" --limit 5

# 2. Download what Speakeasy actually generated
gh run download <RUN_ID> --name generated_provider_code --dir /tmp/generated_from_ci

# 3. Diff against your committed code
diff internal/provider/connection_resource.go /tmp/generated_from_ci/provider/connection_resource.go
diff -rq internal/provider/ /tmp/generated_from_ci/provider/
diff -rq --exclude=.gitattributes --exclude=.gitignore internal/sdk/ /tmp/generated_from_ci/sdk/

# 4. Update your committed files to match the artifact exactly
```

Common gotchas:
- Speakeasy generates separate import aliases for custom vs built-in plan modifiers (e.g., `custom_setplanmodifier` vs `speakeasy_setplanmodifier`)
- Import ordering must be alphabetical within groups
- Plan modifier ordering in the `PlanModifiers` slice may differ from what you expect
- Indentation must match byte-for-byte

For the full procedure, see [CONTRIBUTING.md — Debugging Zero-Diff Failures](CONTRIBUTING.md#debugging-zero-diff-failures).

## Build and Validation Commands

```bash
# Build
go build ./...

# Lint
go vet ./...

# Generate docs (after changing schema descriptions)
go generate ./...
# or: uvx --from=poethepoet poe docs-generate

# Full regeneration pipeline (requires Speakeasy CLI + API key)
uvx --from=poethepoet poe generate-full

# Post-generation patches only
uvx --from=poethepoet poe post-generate
```

## CI Artifacts

The "Test (Full)" workflow produces these artifacts on every PR:

| Artifact | Use for | Download command |
|---|---|---|
| `generated_provider_code` | Debugging zero-diff failures | `gh run download <RUN_ID> --name generated_provider_code --dir /tmp/generated_from_ci` |
| `provider_binaries` | E2E testing with CI-built binaries | `gh run download <RUN_ID> --name provider_binaries --dir ./provider-bin` |
| `api_terraform_spec` | Inspecting the generated OpenAPI spec | `gh run download <RUN_ID> --name api_terraform_spec --dir /tmp/spec` |
| `generated_docs` | Debugging docs generation failures | `gh run download <RUN_ID> --name generated_docs --dir /tmp/docs` |

## Files You Can Safely Edit

- `overlays/terraform_speakeasy.yaml` — Speakeasy overlay
- `scripts/*.py` — Post-generation patch scripts
- `poe_tasks.toml` — Build task definitions
- `internal/planmodifiers/` — Custom plan modifiers (not generated)
- `internal/provider/*_movestate.go` — State migration helpers (not generated)
- `internal/provider/movestate_helpers.go` — Shared movestate helper code (not generated)
- `templates/` — Documentation templates
- `.github/workflows/` — CI workflows
- `test-projects/` — Sample Terraform projects for testing

## Files You Must NOT Edit By Hand

- `internal/provider/*.go` (except `*_movestate.go` and `movestate_helpers.go`) — Generated by Speakeasy
- `internal/sdk/` — Generated by Speakeasy
- `docs/` — Generated from templates by `go generate`
