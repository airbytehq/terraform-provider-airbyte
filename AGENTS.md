# Agent Guidelines for terraform-provider-airbyte

For the full contributor guide (code generation lineage, build commands, release process, documentation, and debugging generation drift), see [CONTRIBUTING.md](CONTRIBUTING.md).

## Key Principle: This Is a Generated Codebase

Most code under `internal/provider/` and `internal/sdk/` is generated by Speakeasy. **Do not hand-edit generated files.** To make durable changes, work at the correct layer (overlay, post-generation script, or custom plan modifier). See [CONTRIBUTING.md — Code Generation Lineage](CONTRIBUTING.md#code-generation-lineage).

## Generation Drift

When a drift CI check fails, download the generated artifacts and compare. See [CONTRIBUTING.md — Debugging Generation Drift Failures](CONTRIBUTING.md#debugging-generation-drift-failures).

Agent-specific pitfalls:
- Speakeasy generates separate import aliases for custom vs built-in plan modifiers (e.g., `custom_setplanmodifier` vs `speakeasy_setplanmodifier`). These must be alphabetical.
- Plan modifier ordering in `PlanModifiers` slices may differ from what you expect — always match the artifact exactly.
- Do not guess or iterate blindly. Download the artifact first.

## Files You Can Safely Edit

- `overlays/terraform_speakeasy.yaml` — Speakeasy overlay
- `scripts/*.py` — Post-generation patch scripts
- `poe_tasks.toml` — Build task definitions
- `internal/planmodifiers/` — Custom plan modifiers (not generated)
- `internal/provider/*_movestate.go` — State migration helpers (not generated)
- `internal/provider/movestate_helpers.go` — Shared movestate helper code (not generated)
- `templates/` — Documentation templates
- `.github/workflows/` — CI workflows
- `test-projects/` — Sample Terraform projects for testing

## Files You Must NOT Edit By Hand

- `internal/provider/*.go` (except `*_movestate.go` and `movestate_helpers.go`) — Generated by Speakeasy
- `internal/sdk/` — Generated by Speakeasy
- `docs/` — Generated from templates by `go generate`
